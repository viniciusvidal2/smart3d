Reconstruction Tool

This tool is used to reconstruct and texturize a mesh using images.

Usage

  1. Press the reconstruction icon in the toolbar.





  2. Check what you want to do.





  3. Fill the fields that are enabled.
  4. Press the "Gears" button if you want to change the configuration.





    • Sparse quality - This is the quality level used to align the images. The greater the quality more RAM and VRAM will be used. This parameter is the main cause to reconstruction fails.
    • Dense quality - This is the quality level used to create the point cloud. The greater the quality more RAM and VRAM will be used. This parameter is the main cause to reconstruction fails.
    • Mesher - Define the mesh algorithm that will be used.
      • Poisson - Create more smooth surfaces.
      • Delaunay - Create more complete surfaces.
    • Use GPU - Define if the GPU is going to be used by the algorithm.
    • Use COLMAP - Force the use of COLMAP algorithm to generate the mesh (instead of MeshRecon).
    • Bounding box type
      • 0 - All the elements in the image will be reconstructed.
      • 1 - Just the elements that appear in all images will be reconstructed.
    • Level of details - The greater the level of details more RAM and VRAM will be used. This parameter is the main cause to reconstruction fails, usually you do not have sufficient VRAM.
    • Disable GPU memory check - Force MeshRecon to use the level of details chosen, regardless of the available VRAM.
    • Data Term - Gmi or Area. Area will create more texture files.
    • Outlier removal - Used to remove pedestrians and other moving objects.
    • Geometric visibility test - A collision test used to place the texture in the right place.
    • Global seam leveling - Used to balance the color difference through the mesh, if the images have a big difference in illumination could cause bad results.
    • Local seam leveling - Used to balance the color difference locally, useful to minimize the divisions created in the texturization preocess.
    • Hole filling - Close holes in the mesh.
    • Keep unseen faces - Keep the faces that are not seen by any of the cameras.
    • Multi-band downscale - Width of frequency bands.
    • Best score threshold - 0.0 to disable filtering based on threshold to relative best score (0.0-1.0).
    • Angle hard threshold - 0.0 to disable angle hard threshold filtering (0.0, 180.0).
    • Force visible by all vertices - Triangle visibility is based on the union of vertices visiblity.
    • Visibility remapping method
      • Push - For each vertex of the input mesh, pull the visibilities from the closest vertex in the reconstruction.
      • Pull - For each vertex of the reconstruction, push the visibilities to the closest triangle in the input mesh.
      • Push-Pull - Combine results from Pull and Push results.
    • Texture side - Output texture size (1024, 2048, 4096, 8192, 16384).
    • Padding - Texture edge padding size in pixel (0-100).
    • Downscale - Texture downscale factor (1, 2, 4, 8).
    • Fill holes - Fill holes in the texture.
    • Use UDIM - Use UDIM UV mapping.
  5. To start processing:
    • If you want to process only one dataset press "OK" to start the process or "Cancel" to abort.
    • If you want to create a queue of reconstructions, press "Add" until you add all data you want to process, then press "Start".
